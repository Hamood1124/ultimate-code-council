# Code Council — File Classification

Classify every file before reviewers run. This prevents wasting tokens on files the user didn't write and ensures deep analysis where it matters.

---

## Three tiers

### 🔵 Deep scan
Full six-reviewer pass. Every line matters.

**Includes:**
- All user-authored source files: components, pages, controllers, services, routes, models, repositories, utilities, hooks
- Config files the user created or edited: `next.config.js`, `vite.config.ts`, `webpack.config.js`, custom `.env` handling
- Scripts written by the user: build scripts, migration scripts, seed files
- Tests written by the user: `*.test.ts`, `*.spec.ts`, `__tests__/`
- Infrastructure-as-code the user wrote: Dockerfiles, `docker-compose.yml`, Terraform, Bicep files

### 🟡 Quick scan
Structural integrity check only. No full reviewer passes. Flag if obviously broken or suspicious.

**Includes:**
- Lock files: `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`, `poetry.lock`, `Pipfile.lock`
- Generated type files: `*.d.ts` files generated by tools, Prisma client output, GraphQL generated types
- Build output: anything in `dist/`, `build/`, `out/`, `.next/`, `__pycache__/`, `bin/`, `obj/`
- Source maps: `*.map` files
- Auto-generated migrations: database migration files generated by an ORM (flag if contents look hand-edited)
- Proto-generated files: `*.pb.go`, `*_pb2.py`, etc.

**Quick scan check:** Does this file look structurally valid? Is there anything that looks hand-edited that shouldn't be? Flag and move on. Do not run reviewers.

### ⚫ Skip entirely
Do not read. Do not analyze. Note in context header that they were excluded.

**Includes:**
- `node_modules/`
- `vendor/` (Go, PHP)
- `venv/`, `.venv/`, `env/` (Python virtualenvs)
- `.git/`
- Binary files: images, fonts, compiled binaries, `*.wasm`
- Log files: `*.log`, `logs/`
- OS files: `.DS_Store`, `Thumbs.db`

## Auto-detection rules

### Detect project type from file tree

| Files found | Project type |
|-------------|--------------|
| `package.json` + `tsconfig.json` + `src/` | TypeScript/Node |
| `package.json` + React imports | React frontend |
| `.csproj` or `.sln` | .NET / C# |
| `requirements.txt` or `pyproject.toml` | Python |
| `go.mod` | Go |
| `Cargo.toml` | Rust |
| `pom.xml` or `build.gradle` | Java/Kotlin |
| SPFx: `package.json` + `sharepoint/` folder | SPFx/SharePoint |
| Electron: `package.json` + `main` pointing to electron | Electron app |

### Detect available tooling from project type

| Project type | Run these tools |
|--------------|----------------|
| TypeScript | `tsc --noEmit`, `eslint` or `biome` |
| Python | `pytest`, `mypy`, `ruff` |
| .NET / C# | `dotnet build`, `dotnet test` |
| Go | `go build ./...`, `go vet ./...`, `go test ./...` |
| Rust | `cargo check`, `cargo test`, `cargo clippy` |
| Any Node | `npm audit` / `pnpm audit` |
| Any Python | `pip-audit` (at Deep depth) |
